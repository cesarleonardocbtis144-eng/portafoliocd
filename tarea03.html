Módulo 01 · Base absoluta

Fundamentos de HTML

HTML (HyperText Markup Language) es el lenguaje de marcado estándar para crear páginas web. No tiene variables ni lógica — es un lenguaje de estructura: le dice al navegador qué es cada cosa (un título, un párrafo, una imagen).

Estructura mínima de todo documento

index.html
<!-- Todo documento HTML5 comienza exactamente así --> <!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"/> <meta name="viewport" content="width=device-width, initial-scale=1.0"/> <title>Mi primera página</title> </head> <body> <h1>¡Hola, mundo!</h1> <p>Esta es mi primera página web.</p> </body> </html>
¿Qué hace cada parte?
<!DOCTYPE html> — Declara HTML5. Siempre va primero.
<html lang="es"> — Raíz del documento. Define el idioma.
<head> — Metadatos invisibles para el usuario.
<meta charset="UTF-8"> — Permite tildes y caracteres especiales.
<meta name="viewport"> — Hace la página responsiva en móviles.
<body> — Todo el contenido visible va aquí.

Anatomía de una etiqueta

anatomia.html
<!-- Etiqueta con apertura, atributos, contenido y cierre --> <p class="intro" id="principal">Texto del párrafo</p> <!-- Etiquetas auto-cerradas — no tienen contenido interno --> <img src="foto.jpg" alt="Descripción"/> <br/> <!-- salto de línea --> <hr/> <!-- línea horizontal --> <input type="text"/> <meta charset="UTF-8"/>

Módulo 02 · Contenido textual

Texto & Tipografía

Las etiquetas de texto son las más usadas. Algunas tienen peso semántico (le dicen al navegador el significado) y otras son puramente visuales. Preferir siempre las semánticas.

Encabezados — jerarquía lógica

encabezados.html
<h1>Título principal — solo uno por página</h1> <h2>Subtítulo de sección</h2> <h3>Subtítulo de subsección</h3> <h4>Nivel 4</h4> <h5>Nivel 5</h5> <h6>Nivel 6 — el más pequeño</h6>

Formato, citas y código

texto.html
<p>Texto en <strong>negrita semántica</strong> y <b>negrita visual</b>.</p> <p>Texto en <em>cursiva con énfasis</em> y <i>cursiva visual</i>.</p> <p><mark>Resaltado</mark> · <del>tachado</del> · <ins>insertado</ins></p> <p>H<sub>2</sub>O &nbsp;·&nbsp; E=mc<sup>2</sup></p> <p>Código en línea: <code>console.log()</code></p> <p>Abrev: <abbr title="HyperText Markup Language">HTML</abbr></p> <pre> Texto preformateado conserva espacios y saltos.</pre> <blockquote> <p>La web es para todos.</p> <footer>— <cite>Tim Berners-Lee</cite></footer> </blockquote> <p>Como dijo: <q>el conocimiento es poder</q>.</p>

Enlaces e Imágenes

El elemento <a> — hipervínculos

<!-- Enlace externo en nueva pestaña --> <a href="https://google.com" target="_blank" rel="noopener">Google</a> <!-- Enlace interno (misma carpeta) --> <a href="/sobre-mi.html">Sobre mí</a> <!-- Ancla — salta a una sección de la misma página --> <a href="#contacto">Ir a Contacto</a> <section id="contacto">...</section> <!-- Correo electrónico --> <a href="mailto:hola@correo.com">Enviar correo</a> <!-- Número de teléfono --> <a href="tel:+529612345678">Llamar</a> <!-- Descarga de archivo --> <a href="guia.pdf" download>Descargar PDF</a>

Imágenes responsivas y <figure>

imagenes.html
<!-- alt es OBLIGATORIO — siempre --> <img src="foto.jpg" alt="Descripción clara" width="400" height="300"/> <!-- Imagen responsiva con srcset --> <img src="img-400.jpg" srcset="img-400.jpg 400w, img-800.jpg 800w" sizes="(max-width:600px) 400px, 800px" alt="Imagen responsiva" loading="lazy" /> <!-- Figura con pie de imagen --> <figure> <img src="grafica.png" alt="Gráfica del semestre"/> <figcaption>Fig. 1 — Resultados del primer semestre.</figcaption> </figure> <!-- Imagen decorativa: alt vacío --> <img src="fondo.png" alt="" role="presentation"/>
Nunca omitas el alt. Si la imagen es decorativa usa alt="". Si es informativa, describe lo que muestra con precisión para lectores de pantalla.

Módulo 07 · HTML5 moderno

Semántica HTML5

HTML5 introdujo etiquetas que describen el propósito del contenido. Esto mejora el SEO, la accesibilidad y la legibilidad del código. La diferencia entre usar <div> para todo vs. usar etiquetas con significado.

semantica.html
<body> <header> <!-- Encabezado del sitio --> <nav> <!-- Menú de navegación --> <ul> <li><a href="/">Inicio</a></li> <li><a href="/tareas">Tareas</a></li> </ul> </nav> </header> <main> <!-- Único por página --> <article> <!-- Contenido independiente --> <header> <h1>Tim Berners-Lee</h1> <time datetime="2025-03-15">15 Mar 2025</time> </header> <p>Contenido del artículo...</p> </article> <section id="lenguajes"> <!-- Sección temática --> <h2>Lenguajes</h2> </section> </main> <aside> <!-- Barra lateral --> <h3>Relacionados</h3> </aside> <footer> <!-- Pie del sitio --> <p>&copy; 2025 Portafolio</p> <!-- Expandible nativo sin JavaScript --> <details> <summary>Más información</summary> <p>Cultura Digital 2 · 2° Semestre IA</p> </details> </footer> </body>
EtiquetaUso correcto
<header>Encabezado del sitio o una sección. Puede haber varios.
<nav>Bloque de navegación principal del sitio.
<main>Contenido principal. Solo uno por página.
<article>Contenido independiente que puede vivir solo (post, noticia).
<section>Agrupación temática que necesita contexto.
<aside>Contenido tangencial: sidebar, anuncios, relacionados.
<footer>Pie de página o sección. Puede haber varios.
<time>Fecha/hora legible por máquinas con atributo datetime.
<details>Contenido expandible/colapsable sin JavaScript.
<summary>Título visible de un <details>.
<figure>Imagen, diagrama o código con pie de nota.
<address>Información de contacto del autor o sitio.

Los atributos globales funcionan en cualquier etiqueta HTML. Son el puente entre HTML, CSS y JavaScript, y la base de la accesibilidad web.

Módulo 12 · Profesionalismo

Buenas Prácticas y Ruta

Escribir HTML correcto no es solo que funcione — es que sea accesible, semántico y mantenible. Estas reglas marcan la diferencia entre código amateur y profesional.

✓ Usa etiquetas semánticas — <header>, <main>, <article> en vez de <div> para todo.

✓ Un solo <h1> por página — La jerarquía h1→h2→h3 debe ser lógica y sin saltos.

✓ Siempre incluye alt en imágenes. Si es decorativa: alt="".

✓ Cada input necesita su label con atributos for/id coincidentes.

✓ Indenta con 2 espacios y usa siempre comillas dobles en atributos.

✓ Usa loading="lazy" en imágenes e iframes fuera de la pantalla inicial.

✓ Nunca uses tablas para layout — son exclusivamente para datos tabulares.
